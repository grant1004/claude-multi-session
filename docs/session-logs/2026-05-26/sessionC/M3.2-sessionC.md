# M3.2 — Add `/multi-session:self-check` worker pre-commit validation command

> Commit `d6c6c6e` · sessionC · 2026-05-26

## 📦 變更摘要 / Change summary
- `plugins/claude-multi-session/commands/multi-session/self-check.md`: New slash command — 6-step Worker self-validation with 4 auto-pass criteria checks

## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Auto-detect pre-commit vs post-commit mode**
   - Why: Workers may run self-check before OR after committing. Pre-commit can't verify commit message; post-commit checks the actual commit.
   - Approach: If HEAD commit message matches `^Mx.y:`, assume post-commit mode (use `git diff --stat HEAD~1`). Otherwise assume pre-commit (use staged + unstaged diff).

2. **Three-tier milestone ID resolution (arg → context → git)**
   - Why: Workers may invoke the command in different ways — explicitly with an ID, during a conversation with a dispatch visible, or after already committing.
   - Fallback chain: explicit arg > dispatch message in conversation > most recent commit message prefix > ask user.

3. **Special allowances for PROGRESS.md and session-logs in file scope check**
   - Why: These files are REQUIRED by workflow rules but never listed in a milestone's "expected files" (they're meta-artifacts, not deliverables). Treating them as "out of scope" would always produce a false fail.
   - Solution: Whitelist `PROGRESS.md` and `docs/session-logs/**` — they always pass scope check regardless of expected files list.

4. **Warnings (⚠️) vs hard fails (❌) distinction**
   - Why: Some issues are informational (missing `<!-- sessionId -->` comment, can't verify commit message pre-commit) while others are blocking (wrong files changed, no atomic log). Conflating them obscures what actually needs fixing.
   - Rule: Only the 4 core auto-pass criteria produce ❌. Everything else is ⚠️.

## ✅ 驗收條件對照 / Acceptance criteria
