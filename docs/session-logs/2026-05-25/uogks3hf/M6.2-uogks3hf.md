## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Kept write-ownership rules as defense-in-depth**
   - Why: Worktree isolation eliminates the physical race condition, but write-ownership (Reviewer-only for 現在進度, Worker-only for own checkbox) remains as a logical constraint. Belt-and-suspenders approach.
   - Alternative considered: Removing write-ownership rules entirely — rejected because merge conflicts at Reviewer merge time still benefit from clear ownership.

2. **Placed Worktree lifecycle section between file-region partitioning and design-decision skip-list**
   - Why: File-region partitioning is the most important invariant; worktree lifecycle is the mechanism that enforces it. Skip-list and abort protocol are operational procedures that build on top of both.

3. **Used `git fetch origin && git rebase main` (not just `git rebase main`) for worker pre-milestone rebase**
   - Why: If the repo has a remote, `git fetch` ensures main is up to date with origin before rebasing. For local-only repos, `git fetch` is a harmless no-op.
