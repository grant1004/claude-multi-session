2. **Rebase target: `main` → `session/<slug>`**
   - Why: Workers no longer rebase onto main. The session branch is the integration target — Reviewer merges worker branches into it, then session branch merges into main at the end.

3. **Commit target: `session/<your-id>` → `worker/<your-id>`**
   - Why: Workers commit to their own worker branch. Added explicit "never directly to main or the session branch" to cover both forbidden targets.
   - Updated merge description: "Reviewer merges to the session branch" (not main).

4. **Common mistakes: updated both branch ref and rebase ref**
   - Why: Two separate mistakes listed — committing to wrong branch and forgetting to rebase. Each needed its own update to match the new model.
