## Change summary
- `tests/validate-templates.sh`: New bash script with 6 structural checks across all plugin template and command files
- `tests/README.md`: Documents what the script checks and how to run it

## Implementation notes
1. **Chose bash over Node/Bun**
   - Why: Zero runtime dependencies, runs anywhere with a POSIX shell (including GitHub Actions runners natively)
   - Alternative considered: Node script with a YAML parser — rejected because frontmatter checks are simple enough with awk/grep

2. **Log-template frontmatter check validates inside code blocks, not at file top**
   - Why: Log templates are documentation files that demonstrate the format; the YAML frontmatter example lives inside a fenced code block, not at line 1
   - Alternative considered: Requiring actual top-level frontmatter on templates — rejected because it would change the template file semantics

3. **Init.md copy-list hardcoded in script**
   - Why: The copy list in init.md is prose, not machine-readable. Hardcoding the 11 expected files is more reliable than parsing natural language.
   - If init.md's list changes, the script's array must be updated manually.
