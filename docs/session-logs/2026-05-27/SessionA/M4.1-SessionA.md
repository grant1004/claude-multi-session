## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Two-tier instead of three-tier pattern**
   - Why: Workers should never ask the user to install tools — they execute milestones autonomously. The audit.md three-tier pattern (try → ask → fallback) is appropriate for Reviewer's interactive audit phase, but Workers need silent fallback.
   - Alternative considered: Three-tier (matching audit.md exactly) — rejected because Workers shouldn't interrupt execution to prompt for tool installation.

2. **Placement of codebase-memory step in Setup (step 3, before set_summary)**
   - Why: Tools should be loaded before the Worker starts any work, but after reading the role/workflow docs (which don't require codebase-memory). Loading before `set_summary` ensures the Worker knows its tool availability before reporting ready.

3. **Responsibility item placed between "Completion report" and "Daily summary"**
   - Why: Code exploration is a per-milestone activity (like completion reports), not a session-level activity (like daily summaries). Placing it here groups it with other per-milestone responsibilities.
