## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Used `exec` instead of subshell launch**
   - Why: `exec` replaces the shell process with `claude`, so the launcher doesn't linger as a parent process. Exit code propagation is automatic.
   - Alternative considered: `claude "$@"; exit $?` — rejected because it keeps an unnecessary shell process alive and requires explicit exit code forwarding.

2. **`set -euo pipefail` for strict mode**
   - Why: Script is short but should fail loudly on any unexpected condition (missing `claude` binary, unset variable). Matches defensive scripting conventions.

3. **Simple positional parse instead of `getopts`**
   - Why: The script only has one flag (`-id`), and it must appear first. `getopts` adds complexity for no benefit. Everything after `-id <name>` is passed through verbatim.
