## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Placed as step 6, after set_summary and before the "Confirm" line**
   - Why: codebase-memory loading is a tool-setup step, logically grouped with other setup steps (reading docs, setting summary). Putting it last among setup steps means the Worker has role context before deciding whether to load optional tools.
   - Alternative considered: inserting earlier (e.g. step 2) — rejected because Workers should understand their role before loading optional tools.

2. **Marked as "optional but recommended" with explicit fallback**
   - Why: codebase-memory MCP may not be available in all environments. Workers must not block on it. Matches the two-tier pattern from ADR-001.
   - Alternative considered: making it mandatory — rejected because it would break onboarding for setups without codebase-memory MCP.
