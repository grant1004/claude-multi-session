# M2.2 — Add GitHub Actions CI workflow for template validation

> Commit `d0def5a` · sessionB · 2026-05-26

## Change summary
- `.github/workflows/validate.yml`: GitHub Actions workflow with 2 validation steps

## Implementation notes
1. **Two-step validation approach**
   - Step 1: Run `tests/validate-templates.sh` (structural checks from M2.1)
   - Step 2: `diff -r` root templates vs plugin source (drift guard)
   - Why separate steps: clearer failure diagnosis — "structure broken" vs "templates drifted"

2. **No dependencies needed**
   - Why: Both checks are pure bash/coreutils. ubuntu-latest has everything.
   - Alternative considered: Adding a Node setup step for potential future JS tests — rejected as premature.

3. **Drift guard uses diff -r with error annotation**
   - `::error::` prefix makes GitHub Actions highlight the failure in the PR UI
   - Keeps the raw diff output visible for debugging which files drifted
