## 📦 變更摘要 / Change summary
- `QUICKSTART.md`: Expanded all 8 steps with macOS/Linux variants. Title changed from "Windows, zero-baseline" to just "Quickstart". Platform-specific steps split with `### Windows (PowerShell)` / `### macOS / Linux (bash/zsh)` headers. Platform-neutral steps marked as such with `sh` code blocks.

## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Split by headers, not tabs**
   - Why: Markdown doesn't natively support tabs (GitHub-style `<details>` or HTML tabs would be fragile). Subsection headers are readable in any renderer.
   - Alternative considered: collapsible `<details>` blocks — rejected because they hide content by default and require HTML inside markdown.

2. **Steps 4, 6, 7 marked platform-neutral instead of duplicated**
   - Why: These steps use `claude` CLI commands or generic `cd`/`mkdir` that work identically on all platforms. Duplicating would add noise without value.
   - Changed code fences from ` ```powershell` to ` ```sh` for these sections.

3. **Step 5 got the heaviest expansion**
   - Why: Launcher install is the most platform-divergent step (different script, different PATH mechanism, chmod requirement). Full subsection with `~/.local/bin` setup, `chmod +x`, and shell config sourcing.

4. **Troubleshooting items split inline**
   - Why: Keeps each issue in one place with platform-specific workarounds side-by-side, rather than duplicating the entire troubleshooting section.
