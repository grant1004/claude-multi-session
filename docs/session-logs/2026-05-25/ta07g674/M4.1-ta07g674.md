## 📦 變更摘要 / Change summary
- `plugins/claude-multi-session/commands/multi-session/status.md`: New slash command — reads PROGRESS.md and prints a concise dashboard with phase, milestone table, counts, and skip list

## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Locked allowed-tools to `Read, Bash(git:*)` only**
   - Why: Status is purely informational. Restricting tools prevents accidental file edits or peer messages from a status check. Matches the acceptance criteria requirement.
   - Alternative considered: Including `Glob` and `Grep` for broader file discovery — rejected because the command only needs to read one file (PROGRESS.md).

2. **Three-state status icons (✅/🔄/⬜) driven by both checkbox and frontmatter**
   - Why: `[x]` alone can't distinguish "completed" from "in-progress". The `in_progress:` frontmatter array (already part of PROGRESS.md's YAML schema) provides the third state without inventing new markup.
   - Alternative considered: Parsing 「註」 text for keywords like "working" — rejected because it's fragile and language-dependent.

3. **Context injection pattern mirrors existing commands**
   - Why: Consistency with audit.md, roll-call.md, init.md. Uses `!` backtick injection for PROGRESS.md existence check so the command can short-circuit with a helpful message before doing any work.
