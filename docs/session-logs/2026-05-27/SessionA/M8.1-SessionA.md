## 🎯 實作要點 / 設計決策 / Implementation notes
1. **New "Branch model" section with ASCII diagram**
   - Why: The three-layer branch model (main → session → worker) is the single most important concept in the new lifecycle. An ASCII diagram makes the merge direction immediately visual.
   - Placed before "Roles at a glance" since it's foundational context for all roles.

2. **State machine: two new steps**
   - `[Create session branch]` after `[Reviewer init]`: `git checkout -b session/<YYYY-MM-DD>-<slug> main`. Created by audit, once per run.
   - `[Finalize]` after `[Cleanup]`: `git checkout main && git merge --no-ff session/<slug>`. User confirms. Only point where main is modified.

3. **All merge/rebase targets changed from main to session branch**
   - `[Create worktrees]`: `worker/<id>` branches from `session/<slug>` (not main)
   - `[Review]`: merge target is `session/<slug>` (not main)
   - `[Pre-next rebase]`: rebase target is `session/<slug>` (not main)
   - `[Cleanup]`: deletes `worker/<id>` branches (not `session/<id>`)
   - PROGRESS.md write strategy: merge integrates into session branch (not main)

4. **Worktree lifecycle: added Finalize subsection**
   - Explains `--no-ff` merge to main with user confirmation
   - Cleanup subsection updated: worker branches deleted, session branch remains until finalize

5. **Pitfalls table: two new entries**
   - "Worker rebases from main instead of session branch" — must rebase from session/<slug>
   - "Finalize without user confirmation" — --no-ff merge requires explicit user confirmation
