## 📦 變更摘要 / Change summary
- `plugins/claude-multi-session/commands/multi-session/upgrade.md`: new slash command for upgrading .claude-multi-session/ templates to match current plugin version

## 🎯 實作要點 / 設計決策 / Implementation notes
1. **Four-category diff classification (unchanged/modified/new/removed)**
   - Why: covers all possible drift scenarios between plugin source and live project templates
   - Alternative considered: only check for modified files — rejected because version upgrades can add or remove template files

2. **Three-option user flow (apply all / review each / cancel)**
   - Why: balances speed (bulk apply) with control (per-file review) and safety (cancel)
   - Follows the same AskUserQuestion pattern used in init.md's git pre-check

3. **Never auto-delete removed files**
   - Why: user may have intentionally added custom files to .claude-multi-session/; only warn and ask
   - In "apply all" mode, removed files are left alone (reported but not deleted)

4. **allowed-tools includes Bash(cp:*) and Bash(ls:*) in addition to dispatch spec**
   - Why: command needs `cp` to copy new files and `ls` to walk directories. Original spec listed only Read, Bash(diff:*), Write, Edit, AskUserQuestion — expanded minimally for functional needs.
