# M3.3 — Add /multi-session:review helper command

> Commit `97b1548` · sessionB · 2026-05-26

## Change summary
- `plugins/claude-multi-session/commands/multi-session/review.md`: New Reviewer helper command (9-step structured review flow)

## Implementation notes
1. **Modeled after dispatch.md's pattern**
   - Why: Consistency across Reviewer helper commands — same AskUserQuestion flow, same fenced-code-block output, same "never writes/sends automatically" principle
   - dispatch.md = 10 steps (generate dispatch), review.md = 9 steps (generate verdict + optional merge)

2. **Per-criterion evidence linking**
   - Why: Forces the review to be evidence-based rather than "looks good to me". Each criterion gets a specific file:line or diff excerpt proving it's met.
   - Trade-off: Longer review output, but much higher signal for Workers on what to fix on fail.

3. **Optional merge as the only write action**
   - Why: Merge is the natural next step after pass. Making the Reviewer switch terminals to do it manually is friction with no safety benefit (the command already asks for explicit confirmation).
   - Guarded by AskUserQuestion — never auto-merges.

4. **Branch auto-detection from `git branch --list 'session/*'`**
   - Why: Reduces Reviewer cognitive load — if only one session branch has commits ahead of main, skip the question entirely.
